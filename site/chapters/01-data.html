Chapter 01 · pretrain / data

Text must become numbers.

The model never sees words. This chapter streams stories, learns a small byte-pair vocabulary, writes token IDs, and turns one window into input and target tensors.

1. Prepare a tiny practice dataset

TinyStories provides short synthetic stories. We stream 200 training stories for this practice pass, plus the preset’s fixed validation split, keeping the feedback loop smaller than a full run. The complete quick preset comes after you understand the pipeline.

Run · practice pass

Download, split, tokenize, encode

The output directory is deliberately named practice so it will not collide with the full quick run.

source .venv/bin/activate
macllm prepare --preset quick --max-stories 200 --output data/prepared/practice
Expected: tokenizer.json, train.bin, valid.bin, and dataset.json under data/prepared/practice/.
Why BPE?

Byte-pair encoding begins with bytes and repeatedly merges common neighboring pieces. It can represent any input while giving frequent patterns such as the or ing compact IDs.

2. The knobs live in one place

config.py contains model shape and training choices, not learning logic. Presets let us change experiment size without scattering constants through the code.

Model shape

Vocabulary, vector width, layer count, attention heads, context length, and numerical constants define one transformer.

Training recipe

Batch size, steps, learning rate, evaluation cadence, data limits, seed, decay, and clipping define one run.

Three experiment sizes

Quick proves the loop; standard is the practical course model; overnight trades much more waiting for capacity.

Validate and save

Configurations are checked and serialized beside weights so a checkpoint can reconstruct the exact model.

3. Follow a story into a batch

The data module has one path from text documents to compact integers and one tiny dataset class that samples random training windows.

Obtain documents

Local text is split at blank lines; TinyStories is streamed lazily; documents are written with clean separators.

Learn the vocabulary

A byte-level BPE tokenizer learns frequent pieces from training text only and saves its rules.

Encode without filling memory

Batches of documents become unsigned 32-bit IDs. An end token separates stories before bytes are flushed to disk.

Build train and validation

Separate held-out text gives an honest test. Temporary raw text is removed after both splits are encoded.

Make the prediction shift

Memory mapping avoids loading every token. A random window of length N+1 becomes N inputs and the same window shifted one place as targets.

4. Inspect the actual tokens

The input row stops one token earlier; the target row starts one token later. Every column is a supervised next-token question, and the causal mask later prevents answers from leaking left.

Run · inspect

See your trained tokenizer’s pieces

This is more truthful than splitting on spaces: BPE chooses pieces from the corpus it just read.

macllm inspect --data data/prepared/practice --text "Once upon a little time"
Expected: parallel INPUT and TARGET rows, shifted by exactly one token.
