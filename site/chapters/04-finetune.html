Chapter 04 · shape behavior

Same objective. Better-shaped examples.

Fine-tuning still predicts the next token. The difference is the dataset: prompts describe the task, completions demonstrate the desired behavior, and a mask makes only completion errors update the model.

1. Behavior starts in the dataset

Each JSON line is one independent example. The prompt is context; the completion is the answer style to learn. A small set of excellent, varied examples is more educational than a large noisy dump.

Loss mask

The model reads prompt and completion together, but prompt positions receive mask value 0 and completion positions receive 1. The prompt guides the prediction without being treated as text the model must reproduce.

Inspect · teaching data

Read the examples before training

Look for consistent prompt shape, concise target style, variety, and no accidental contradictions.

sed -n '1,5p' data/our-model/train.jsonl
Expected: one valid JSON object per line, each with string prompt and completion fields.

2. Read the full fine-tuner

Our small model is cheap enough to update every parameter. The file reuses checkpoint loading, the existing tokenizer, MLX gradients, AdamW, clipping, metrics, and checkpoint saving.

Validate examples

JSONL is read line by line; malformed or empty datasets fail early with useful file and line context.

Join, shift, and mask

Prompt and completion IDs are joined, truncated, shifted into input/target rows, and marked so only completion targets count.

Average selected errors

Token-level cross-entropy is multiplied by the mask and divided by the number of selected tokens.

Resume from learned weights

Load an unquantized checkpoint, switch to training mode, load data, seed sampling, create AdamW, and prepare metrics.

Update and save

Sample examples, compute masked loss and gradients, clip, update, report, then save a complete derived checkpoint.

3. Full fine-tune our model

Use greedy decoding before and after so sampling randomness cannot masquerade as learning. The quick checkpoint works for the course; the 57M standard checkpoint gives a more meaningful behavior change.

Run · after quick pretraining

Save a baseline, tune, repeat exactly

The first and last commands differ only by checkpoint path. The shell’s $'…' quoting turns \n into the real newline used in the training prompts.

macllm generate --checkpoint runs/quick --prompt $'Write a tiny story about a patient robot.\nStory:' --temperature 0

macllm finetune --checkpoint runs/quick --data data/our-model/train.jsonl --output runs/quick-story-tuned --steps 300

macllm generate --checkpoint runs/quick-story-tuned --prompt $'Write a tiny story about a patient robot.\nStory:' --temperature 0
Compare: structure, brevity, and story style—not whether one random wording feels prettier.

4. QLoRA changes what must learn

A 4B model is too large for comfortable full fine-tuning in this teaching loop. QLoRA freezes a 4-bit base and trains small low-rank matrices in selected layers. You learn the trade-off without reimplementing MLX-LM.

LoRA

Instead of changing a large weight matrix W, learn two thin matrices A and B whose product adds a small update. “QLoRA” keeps the frozen base quantized while adapters learn at higher precision.

Run · optional 4B lab

Train adapters on eight layers

The first run downloads roughly a few gigabytes. This command belongs to the installed mlx-lm library; the course does not copy that library’s much larger implementation.

mlx_lm.lora --model mlx-community/Qwen3-4B-4bit --train --data data/qwen --mask-prompt --iters 300 --batch-size 1 --num-layers 8 --max-seq-length 1024 --grad-checkpoint --adapter-path runs/qwen3-4b-adapter
Then run: mlx_lm.chat --model mlx-community/Qwen3-4B-4bit --adapter-path runs/qwen3-4b-adapter.

Checkpoint · test retention too

Ask held-out task questions and unrelated questions. A useful fine-tune improves the target behavior without making every answer imitate one memorized completion.
