1. A neuron is a small numeric function

A neuron receives numbers, multiplies each by a learned weight, adds them with a bias, and passes the result through an activation function. The activation decides how much signal continues. Nothing in this step understands language; useful behavior emerges after many weights learn together.

Neuron

input × weight → add → activate. A weight says how strongly one input matters. A bias shifts when the neuron activates. During training, both are parameters that can change.

2. Layers turn many tiny functions into a network

The output numbers from one layer become the input numbers for the next. This small network uses two inputs and two hidden layers; our transformer uses wider vectors and specialized attention layers, but forward prediction and backward learning follow the same principle.

3. A transformer is a neural network built for sequences

For a batch B, sequence length T, and model width D, the main stream has shape [B, T, D]. Each token is one D-number vector. Attention lets positions exchange information; a feed-forward neural network transforms each position; residual additions keep the stream shape unchanged.

Decoder-only

This model reads only the prefix at each position and predicts what follows. “Decoder” here means causal self-attention, not a mysterious reverse operation.

Embedding

Looks up a learned D-number input vector for each token ID.

Attention

Lets each token gather a weighted mixture of visible earlier token information.

Feed-forward neurons

Applies the same larger gated neural network independently to every token position.

Output logits

Decodes each final vector into one score per possible next token.

4. Attention is controlled communication

Queries ask what a position needs. Keys advertise what positions contain. Values carry the information. The causal mask removes future positions; RoPE makes order visible by rotating query and key coordinates according to position.

Create projections

Queries get all heads. Grouped-query attention uses fewer key/value heads, saving memory while retaining multiple query perspectives.

Split into heads

The input is projected into query, key, and value roles, then reshaped to [batch, heads, time, head_dim].

Add position and cache

RoPE encodes token order. During generation, old keys and values are reused instead of recomputed.

Mask, mix, merge

Scaled dot-product attention performs the communication, then head outputs are reshaped and projected back to model width.

5. One block: normalize, update, add

The code stays short because MLX provides the numerical building blocks. What matters is their arrangement: pre-normalization stabilizes each update and residual addition preserves the previous representation.

SwiGLU computation

One projection creates a SiLU gate, another creates content, multiplication selects useful features, and the final projection returns to D.

Transformer block

Attention and feed-forward each contribute one update through a residual +.

Complete language model

6. Prove the two essential properties

The test creates a tiny model, checks output and cache shapes, then changes future tokens and verifies that earlier logits do not change. That second assertion is direct evidence that the causal mask works.
