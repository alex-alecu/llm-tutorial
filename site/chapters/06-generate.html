1. Follow one token around the complete loop

Inference never updates weights. It repeatedly applies the learned network. The first pass, called prefill, processes every prompt token and creates a cache for each transformer layer. Later passes process only the newest token and reuse that cache.

What is kept?

The loop keeps generated token IDs and, inside every layer, cached key and value vectors for earlier positions. It does not keep old activations for backpropagation because inference does not compute gradients.

2. Logits become a decision

At temperature 0, generation chooses the largest logit: deterministic greedy decoding. Above 0, logits are divided by temperature and sampled. Lower values sharpen preferences; higher values flatten them and increase variety.

Inference

Running a trained model without gradient updates. “Generation” is autoregressive inference for text; each result becomes part of the next input.

Temperature 0

Best for before/after experiments because the same weights and prompt give the same answer.

Temperature 0.8

Useful for varied story samples; a fixed seed reproduces the same random choices.

Maximum new tokens

A safety limit on generated length, not a promise that output will be that long.

EOS

The learned end-of-sequence token lets the model stop naturally before the limit.

4. First, confirm the smoke checkpoint runs

Expect poor text from 20 training steps. This test asks only whether the saved checkpoint can reconstruct and generate.

Run · pipeline proof

Generate 80 new tokens

Using temperature 0 makes the result repeatable.

macllm generate --checkpoint runs/practice-smoke --prompt "Once upon" --max-new-tokens 80 --temperature 0
Expected: probably nonsense, but no loading or shape error. Pipeline correctness and language quality are different claims.

5. Compare real checkpoints with a fixed sheet

Use multiple prompts and keep decoding settings constant. Loss measures broad next-token fit; fixed samples reveal concrete behavior; neither replaces the other.

Run · quick checkpoint

Three reproducible story starts

Different seeds intentionally explore different samples while keeping each individual sample repeatable.

macllm generate --checkpoint runs/quick --prompt "Once upon a time" --seed 1
macllm generate --checkpoint runs/quick --prompt "Lily found a strange box" --seed 2
macllm generate --checkpoint runs/quick --prompt "The small robot was afraid" --seed 3
Record: repetition, sentence closure, consistency, and prompt continuation. Do not expect broad knowledge from TinyStories.

Checkpoint · the model is narrow on purpose

A TinyStories checkpoint can demonstrate real next-token learning, attention, optimization, fine-tuning, quantization, and inference. It is not a general assistant, and that honest boundary is part of the lesson.
