Chapter 03 · pretrain / optimization

Let loss reshape the weights.

Each step samples token windows, predicts every next token, computes cross-entropy, backpropagates gradients, clips extreme updates, and lets AdamW adjust the model.

Runa 20-step smoke training job
Diagnosetraining and validation loss together

1. The five-line idea inside the full loop

The implementation adds evaluation, metrics, speed reporting, compilation, and safe checkpoints, but its learning core is still: batch → loss and gradients → clip → optimizer update → evaluate.

Cross-entropy

For each position, the model assigns probabilities to the vocabulary. Cross-entropy penalizes low probability on the correct next token. Averaging across batch and time gives one scalar loss.

Schedule and objective

Warmup grows the learning rate gently; cosine decay reduces it later. Cross-entropy compares logits with token targets.

Closed-book evaluation

Validation batches pass through the same loss without optimizer updates, measuring generalization to held-out stories.

Create the learning state

Load tokenizer, seed randomness, initialize weights and AdamW, ask MLX for value plus gradients, then compile the repeated step.

Repeat updates

Random batches flow through the compiled step. Evaluation happens at the first, scheduled, and final updates.

Measure and recover

Metrics record loss, perplexity, throughput, memory, and time. Each report saves a checkpoint; interruption saves the latest model too.

2. Watch learning begin

Twenty steps are not enough to learn language. That is the point: a smoke run cheaply proves data, model, gradients, optimizer, evaluation, metrics, and checkpoint saving all connect.

Run · about a few minutes

Train on the practice tokens

Run directories are never overwritten. If you already used this name, choose a fresh suffix.

macllm train --preset quick --data data/prepared/practice --steps 20 --output runs/practice-smoke
Expected: reports at steps 1 and 20, plus weights, tokenizer, config, run metadata, and metrics in runs/practice-smoke/.

3. A checkpoint is a runnable model

Weights alone are ambiguous. Saving the tokenizer and configuration beside them makes the directory self-contained; writing weights to a temporary name before replacement avoids leaving a half-written final file.

Save atomically

Write weights, rename them, save config, copy tokenizer, and optionally record run status.

Reconstruct exactly

Load config, create the same shape, apply quantization structure if recorded, load weights, realize tensors, and switch to evaluation mode.

4. Read both loss lines

Training loss asks how well the model fits batches it learns from. Validation loss asks how well that skill transfers. Both falling and staying reasonably close is healthy; training falling while validation rises is overfitting.

Run · visual report

Build and open the local dashboard

The report reads JSONL metrics and creates one self-contained HTML file.

macllm dashboard --run runs/practice-smoke
open runs/practice-smoke/dashboard.html
Expected: a chart with only two points in this smoke run. The full run makes the curve useful.

Map values to pixels

Loss values are normalized into SVG coordinates without a chart dependency.

Read metrics

JSON lines become train, validation, step, and latest-stat collections.

Write one portable page

HTML, CSS, SVG, and current statistics are combined into a report beside the checkpoint.

5. Complete the quick model

Now repeat preparation with the full quick data limit and let all 1,500 steps run. This is the first checkpoint where generation quality is worth judging.

Run · complete quick preset

Prepare and pretrain from random weights

Start this only after the smoke run works. Keep the Mac powered and awake.

macllm prepare --preset quick
macllm train --preset quick
Expected: the validation loss should fall substantially. Actual time depends on measured tokens per second, not a promise on the page.

Checkpoint · separate evidence from hope

A nice random sample is not proof. Keep validation loss, fixed prompts, temperatures, and seeds when comparing checkpoints. The generation chapter gives the exact protocol.
