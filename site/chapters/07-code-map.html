1. One command, one learning concern

The split keeps the code readable: CLI parsing is separate from data, model math, training, fine-tuning, quantization, generation, reports, and environment checks.

doctor

doctor.py checks this Mac and environment.

prepare + inspect

data.py prepares IDs; a small CLI branch displays the shift.

train

training.py combines data, model, loss, optimizer, metrics, and checkpoints.

finetune

finetune.py continues learning with masked prompt/completion examples.

quantize

quantization.py replaces learned matrices with grouped low-bit forms.

generate

generation.py performs cached autoregressive decoding.

dashboard

dashboard.py turns JSONL metrics into a local visual report.

shared foundations

config.py, model.py, and checkpoint.py define reusable state.

4. Tests are executable explanations

The final practice command runs these five files. Each test states one invariant the learning pipeline must preserve, so their complete source belongs in the course too.

Preset progression

Experiment sizes stay ordered and every head/layer shape divides cleanly for tensor operations.

Next-token shift

A known integer stream proves that each target is exactly one position ahead of its input.

Shapes, cache, causality

A tiny transformer produces vocabulary logits, extends its cache, and never lets changed future tokens alter earlier logits.

Prompt masking

A miniature tokenizer and example prove that prompt positions are ignored while completion positions contribute loss.

Readable metrics

Two synthetic records prove that the generated report contains current loss, a validation label, and an SVG line.
