Before the terminal

The small vocabulary behind every LLM.

You do not need machine-learning experience. Learn these objects once, then watch them reappear in the data, model, training, fine-tuning, quantization, and generation code.

Token
A reusable piece of text represented by an integer—not necessarily a complete word.
Vocabulary
The fixed table of token pieces the tokenizer can encode and decode.
Tensor
A multidimensional array holding token IDs, vectors, scores, weights, or gradients.
Parameter
One learned number inside the model. A 57M model has roughly 57 million of them.
Context
The tokens currently visible to a prediction; this course uses 256- or 512-token windows.
Logit
An unnormalized score for one possible next token. Larger means more preferred.
Loss
A single error score. Training tries to lower it; validation checks whether learning transfers.
Gradient
The direction each parameter should move to reduce the current loss.
Optimizer
The rule that converts gradients into careful weight updates; here it is AdamW.
Checkpoint
Saved weights, model configuration, and tokenizer: everything needed to run again.
Fine-tuning
Continuing next-token training on a small dataset shaped around a desired behavior.
Inference
Using frozen weights to predict or generate, without backpropagation or updates.

The loop to remember: tokens go in → logits come out → loss measures error → gradients point downhill → the optimizer changes weights.

03

Quantize the result

Fewer bits. Smaller model. Measured trade-off.

Quantization replaces precise floating-point weights with a small set of levels. The model becomes lighter, but each weight is now an approximation.

Your first real run

From zero to generated text.

Run these four commands from the repository root. Start small; understanding compounds.

  1. 01Enter the environmentUse the installed local dependencies.
    source .venv/bin/activate
  2. 02Prepare storiesStream text, train BPE, write token IDs.
    macllm prepare --preset quick
  3. 03Train the modelAbout 8M weights for the first complete pass.
    macllm train --preset quick
  4. 04Ask it to continueYour checkpoint generates locally on Metal.
    macllm generate --checkpoint runs/quick --prompt "Once upon a time"

Keep your conclusions honest

Three rules for learning from the experiment.

1

Small is not general.

TinyStories can teach coherent simple English. It cannot create a broad assistant from laptop-scale pretraining.

2

Validation beats vibes.

A pretty sample may be luck. Track held-out loss and fixed prompts across checkpoints.

3

Change one variable.

Keep data, seed, and prompts fixed when comparing model size, fine-tuning, or quantization.
