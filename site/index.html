First, one idea

An LLM guesses what comes next.

Give it text. It guesses the next piece. During training, each wrong guess changes millions of stored numbers a little.

1Text“A tiny robot”
2Piecestokens: 41, 92, 8
3Modelweights shape the guess
4Next piece“learned”
Token

A small piece of text stored as a number.

Weight

One learned number. Together, weights store patterns.

Training

Make a guess, score it, then adjust the weights.

Meet the technical words when you are ready
Tensor
A grid of numbers used by the model.
Parameter
The technical name for a learned weight.
Logit
A score for one possible next token.
Loss
One number that says how wrong a prediction was.
Gradient
A hint for how each weight should change.
Optimizer
The rule that applies those changes.
Checkpoint
A saved model you can run later.
Inference
Using the model without changing its weights.

That is enough to begin. Each chapter introduces new terms only when you need them.

03

Quantize the result

Fewer bits. Smaller model. Measured trade-off.

Quantization replaces precise floating-point weights with a small set of levels. The model becomes lighter, but each weight is now an approximation.

Your first real run

From zero to generated text.

Run these four commands from the repository root. Start small; understanding compounds.

  1. 01Enter the environmentUse the installed local dependencies.
    source .venv/bin/activate
  2. 02Prepare storiesStream text, train BPE, write token IDs.
    macllm prepare --preset quick
  3. 03Train the modelAbout 8M weights for the first complete pass.
    macllm train --preset quick
  4. 04Ask it to continueYour checkpoint generates locally on Metal.
    macllm generate --checkpoint runs/quick --prompt "Once upon a time"

Keep your conclusions honest

Three rules for learning from the experiment.

1

Small is not general.

TinyStories can teach coherent simple English. It cannot create a broad assistant from laptop-scale pretraining.

2

Validation beats vibes.

A pretty sample may be luck. Track held-out loss and fixed prompts across checkpoints.

3

Change one variable.

Keep data, seed, and prompts fixed when comparing model size, fine-tuning, or quantization.
